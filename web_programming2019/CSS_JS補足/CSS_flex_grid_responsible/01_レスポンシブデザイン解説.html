5分でわかるレスポンシブ対応

WebサイトをPC、タブレット、スマートフォンなど、様々な画面サイズに合わせて表示させるレスポンシブ対応について解説します。

概要

  • **ビューポート（viewport）**で画面の表示領域を制御します。
  • 画面サイズに応じてCSSを切り替えるには**メディアクエリ（@media）**を使います。
  • 画面幅によってCSSファイルを別々に読み込む場合は**@import**やHTMLの<link>タグを使います。
---

レスポンシブ対応とは？

レスポンシブ対応とは、PC、タブレット、スマートフォンなどの異なる画面サイズに合わせて、Webサイトのレイアウトを柔軟に調整することです。

PCでは横長、スマホでは縦長のレイアウトで表示されるサイトを見たことがありますよね。このレスポンシブ対応には、「ビューポート」と「メディアクエリ」という2つの重要な要素が必要です。

---

ビューポートとは？

ビューポートとは、Webサイトを表示するための領域のことです。ビューポートを指定することで、ブラウザに「この画面領域で表示してね」と指示します。

ビューポートの指定方法

レスポンシブ対応を行う場合、HTMLの<head>タグ内に以下のように記述します。

<meta name="viewport" content="width=device-width, initial-scale=1.0">

contentの中には2つの重要なプロパティがあります。

  • width=device-width: 表示するデバイスの画面幅に合わせて、ビューポートの幅を調整します。
  • initial-scale=1.0: ページが読み込まれたときの初期ズーム倍率を1.0倍（等倍）に設定します。

もしこの設定がないと、モバイルデバイスのブラウザはPCサイトを縮小して表示しようとします。 その結果、文字が小さくなり、ユーザーが拡大しないと読めなくなってしまいます。

---

メディアクエリとは？

メディアクエリとは、表示されている画面サイズに応じて適用するCSSスタイルを切り替える機能です。

CSSファイル内で@mediaルールを使うことで、画面の幅を条件にスタイルを適用できます。

PC表示のメディアクエリ

PCのような大画面にスタイルを適用する場合、画面の最小幅を指定するmin-widthを使います。

@media screen and (min-width: 1024px) { p {color: red;} }

この例では、画面幅が1024px以上になると、pタグの文字色が赤に変わります。この画面幅の境界値をブレイクポイントと呼びます。

スマホ表示のメディアクエリ

スマホのような小さい画面にスタイルを適用する場合、画面の最大幅を指定するmax-widthを使います。

@media screen and (max-width: 767px) { p {color: blue;} }

この例では、画面幅が767px以下になると、pタグの文字色が青に変わります。

タブレット表示のメディアクエリ

タブレットのように中間の画面サイズにスタイルを適用する場合、min-widthとmax-widthをandで繋げます。

@media screen and (min-width: 768px) and (max-width: 1023px) { p {color: green;} }

この例では、画面幅が768pxから1023pxの間にのみ、pタグの文字色が緑に変わります。

---

メディアクエリの記述順

メディアクエリには「デスクトップファースト」と「モバイルファースト」の2つの記述方法があります。

デスクトップファースト

大きい画面（PC）のスタイルをデフォルトにし、小さい画面（タブレット、スマホ）のスタイルで上書きしていく方法です。

/* PC（デフォルト） */ p {color: red;} /* タブレット */ @media screen and (max-width: 1023px) { p {color: green;} } /* スマホ */ @media screen and (max-width: 768px) { p {color: blue;} }

モバイルファースト

小さい画面（スマホ）のスタイルをデフォルトにし、大きい画面のスタイルをメディアクエリで追加していく方法です。

/* スマホ（デフォルト） */ p {color: blue;} /* タブレット */ @media screen and (min-width: 768px) { p {color: green;} } /* PC */ @media screen and (min-width: 1024px) { p {color: red;} }

現在はモバイル端末からのアクセスが増えているため、**モバイルファースト**の記述方法が主流となっています。

1つのCSSファイル内でデスクトップファーストとモバイルファーストを混在させると、コードが複雑になり管理が難しくなるので注意しましょう。

---

別ファイルで画面幅ごとのスタイルを指定したい場合

メディアクエリをCSSファイル内で記述するのではなく、画面幅ごとにCSSファイルを分けて管理することもできます。

HTMLで読み込むCSSを変更する方法

<link>タグのmedia属性を使って、画面サイズに応じたCSSファイルを読み込みます。

<link rel="stylesheet" href="mobile.css"> <link rel="stylesheet" href="tablet.css" media="screen and (min-width: 768px)"> <link rel="stylesheet" href="pc.css" media="screen and (min-width: 1024px)">

CSSで別ファイルを読み込む方法

メインのCSSファイル内で@importルールを使って、メディアクエリの条件付きで別のCSSファイルを読み込みます。

/* main.css */ @import url("./tablet.css") screen and (min-width: 768px); @import url("./pc.css") screen and (min-width: 1024px);

この方法の場合、HTMLではメインのCSSファイルのみを読み込みます。
